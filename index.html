Scalapress

Scalapress is an extensible CMS platform for brochure, ecommerce, blog and directory style sites. Built in the Scala language and using leading JVM frameworks, it is simple to extend by writing your own plugins. Scalapress is a great fit for anyone who is vested in the JVM sphere but wants the power of a product like Drupal or Wordpress.

Ready

Scalapress comes configured with common functionality such as blog posts, folder/page layout, HTML content editors, image/ media galleries. To get up and running is as simple as deploying the app.

Plugin

Scalapress is based on a flexible plugin architecture. The platform is split into several component types such as sections and widgets and anyone can add a new component by implementing the correct interfaces for that component. Scalapress will detect plugins at runtime and make them available in the appropriate places. There are already over 20 plugins available in the plugin browser.

Deploy

Scalapress is a standard "Java" app in the sense that it is a normal deployable WAR that runs in any servlet container such as Tomcat or Jetty. This means Scalapress is familar to anyone who has deployed on the JVM before and there are no bespoke configurations required to get up and running.

Deploying scalapress »
Why would you want to use Scalapress? It's been proven in production - 100's of sites run on Scalapress. See the examples pages for more details. It's easily extensible. Anyone who can write Scala code can write a plugin. You don't need to deeply integrate with the platform as Scalapress will detect your plugins automatically on the classpath. It's high performance. Business friendly Apache license. There are plugins available for many different requirements - ecommerce, mailing lists, payment processors. It offers a highly flexible templating model that mean non developers can integrate with the features through easy to remember tags.
